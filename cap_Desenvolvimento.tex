\section{Prática 3: Uso de chaves/botões e controladores de fluxo(Condicionais)}
\label{pratica3}

Esta prática é destinada, com relação aos tópicos de programação, a introduzir condicionais.

Os objetivos desta prática são: 

\begin{itemize}
	\item Aprender a utilizar botões para enviar sinais digitais para a placa Galileo.
	\item Aprender o uso de condicionais para controlar um programa.
	\item Criar um circuito juntamente com a placa Galileo capaz de acender 4 LED's utilizando um botão para enviar um sinal para a placa de forma que o seguinte padrão seja seguido:
		\subitem No começo da execução do código, os 4 LEDs estarão piscando conjuntamente.
		\subitem Ao se apertar o botão, apenas o primeiro LED piscará, enquanto os outros estarão apagados.
		\subitem Após isso, caso o botão seja apertado, apenas o segundo LED piscará, enquanto os outros estarão apagados.
		\subitem O mesmo padrão seguirá até apenas o quarto LED piscar.
		\subitem Caso, enquanto o último LED estiver piscando, o botão seja apertado, todos os LED voltam a piscar conjuntamente.
		\subitem Caso o botão seja apertado, o padrão supracitado se repetirá.
\end{itemize}

\section{Prática 4: Uso de laços de repetição}
\label{pratica4}

Esta prática é destinada, com relação aos tópicos de programação, ao uso de laços de repetição.

Os objetivos desta prática são: 

\begin{itemize}
	\item Criar um circuito juntamente com a placa Galileo com 4 LED's um botão e um potenciômetro.
	\item Utilizando os componentes supracitados, devem-se ser criados 3 comportamentos distintos no acender e apagar dos LEDs.
	\item Usar o potenciômetro como divisor de tensão( conectar o 5V da placa na pino da direita do potenciômetro, GND na pino da esquerda e a pino centra na entrada analógica A0)
	\item Utilizar o botão para selecionar o comportamento a ser executado.
	\item \textbf{Comportamento 1}: Piscar 1 e apenas um dos 4 LEDs ao depender do valor de tensão lido na porta A0. Os outros 3 LEDs devem permanecer apagado.
	\item \textbf{Comportamento 2}: Piscar os LEDs sequencialmente( pisca LED 1, outros apagados; pisca LED 2, outros apagados; pisca LED3, outros apagados), com velocidade de piscar dependente do valor lido no potenciômetro.
	\item \textbf{Comportamento 3}: Comportamento a ser escolhido pelo grupo. Esse comportamento deve, obrigatoriamente, ser distinto dos dois primeiros comportamentos citados e diferente dos comportamentos criados por outros grupos.
	
\end{itemize}

\section{Prática 5: Uso de vetores, shift register e tipos variados de dados}
\label{pratica5}

Esta prática é destinada, com relação aos tópicos de programação, ao uso de vetores.

Os objetivos desta prática são: 
\textbf{Objetivo 1:}

\begin{itemize}
	\item Criar um circuito juntamente com a placa Galileo com 8 LED's, 8 resitores e um registrador de deslocamento (Shift Register).
	\item Utilizando apenas 3 pinos digitais, o circuito deve ser capaz de acender os 8 LED's.
	\item Para tanto, deve ser utilizado um registrador de deslocamento com pinos propriamente conectados.
	\item Deve ser utilizado um vetor de 8 inteiros para simbolizar um byte que será transmitido pelo registrador de deslocamento. O número simbolizado pelo vetor representa, na forma binaria, quais LED's estarão acessos e quais estarão apagados. O número binário resultante do vetor deve ser convertido num número decimal para se propriamente transmitido pela função shiftOut.
	\item Quais LEDs estarão acessos ou apagados será uma escolha da equipe, entretanto não será permitido a escolha de todos acessos ou todos apagados.  
\end{itemize}

\textbf{Objetivo 2:}

\begin{itemize}
	\item Utilizando o mesmo circuito criado para o objetivo 1, faça um código com o qual os LEDs realizem uma contagem binária. No começo da contagem, todos LEDs estarão apagados, e, no final da contagem que se dará no número 255 ($ 255 = 2^{8} - 1$), todos os LEDs estarão acessos.
	\item Todos os números da contagem devem ser mostrado no monitor serial, na base decimal e na base binária. 
\end{itemize}
